# ❓ Frequently Asked Questions (FAQ)

Welcome to the **45-Days-Python-Development-Challenge** FAQ! 🐍🔥  
This repo is a learning + building journey — so questions about setup, contributions, and structure are common.

---

## 🙋 General

### **Q: Who can participate?**

Anyone! ✅ Beginners, intermediate developers, and experienced engineers are all welcome.

### **Q: Is this only for India / NSoC'26 participants?**

Nope. 🌍 It’s open to everyone. NSoC'26 is one of the community programs collaborating with the repo — the learning challenge is broader.

### **Q: Do I need to code every day for 45 days?**

Ideally yes, but consistency > perfection. 💪 If you miss a day, continue from where you left off.

## 🛠️ Setup & Running Code

### **Q: Which Python version should I use?**

Use **Python 3.10+** (3.11/3.12 is great). 🐍

### **Q: Do I need to install dependencies?**

Many scripts run with standard library only, but some lessons/projects may require packages.  
If a folder/project needs dependencies, it should include instructions or a `requirements.txt`.

### **Q: My script fails on Windows/macOS/Linux. What should I do?**

- 🔍 Re-check the exact command (PowerShell vs bash differs)
- 🧪 Ensure you activated your virtual environment (`venv`)
- 📁 Run from the correct folder (relative paths matter)
- 🧾 Open an issue with error output + OS + Python version

## 🤝 Contributing

### **Q: How do I contribute?**

Follow `CONTRIBUTING.md` 🚀 (fork → branch → PR).

### **Q: Can I open a PR without an issue?**

Best practice here is **issue-first** 🧩 (especially during programs like NSoC'26).  
For tiny fixes (typos), maintainers may still accept PRs, but issues keep work organized.

### **Q: What is a good first contribution?**

- 📝 Improve a `lesson.md`
- 🐛 Fix a broken example or edge case
- ✅ Add a small test
- 🧹 Improve folder naming/consistency

---

## 🔐 Security

### **Q: I found an API key / secret in the repo. What now?**

Please report it privately using `SECURITY.md`. 🛡️  
Do **not** repost the secret in issues, screenshots, or PR comments.
